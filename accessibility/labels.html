Form elements include clearly associated labels

Form inputs such as textboxes, select lists, radio buttons and checkboxes must be labelled so that users can understand what is expected to be entered.

This helps users with cognitive, language or learning disabilities to input or select the necessary information in your service and reduce errors from being made.

This is a requirement of:

WCAG 1.3.1 - Information and relationships (Level A)
WCAG 3.3.2 - Labels or Instructions (Level A)

You should ensure that form inputs are correctly associated with a corresponding label by using the “for” attribute which matches the id of the input.

<label for="first-name">First name</label>
<input id="first-name" name="first-name" type="text">

In some instances a label can be visually hidden, these are still available to screen readers but hidden from sighted users.

This may be appropriate for search fields where other indicators indicate to the user what the field is, for example placeholder text and appropriate button icons or text.

If using GOV.UK Frontend you can use the following css class to achieve this:

<label class="govuk-visually-hidden" for="search-input">Search the website</label>
Do
All form inputs must have a corresponding label element. Use labels above inputs to help users remember what they are filling in.
Don’t
Don’t use placeholders on their own. Placeholder text will dissapear when typing starts and users might forget what the field is for.
