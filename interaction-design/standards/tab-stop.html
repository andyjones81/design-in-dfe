Design interactions in a logical order

Users must be able to interact with a service using only a keyboard. This is because:

  • using a mouse is impossible for people who can’t see the pointer
  • some people don’t have the ability to control a pointing device like a mouse
  • many people without disabilities prefer to use a keyboard

This is a requirement of:

  • WCAG 2.1.1 - Keyboard (Level A)

Good tab stop design allows keyboard users to:

  • access all of the services interactive elements in a predictable order
  • identify which element currently has the input focus.
  • poorly-designed tab stops create high-impact accessibility problems
Do
Use visual hierarchy in HTML to flow left to right top to bottom.
Don’t
Don’t confuse the logical order .

Example of testing tab stop on a webpage

You can use tools such as Accessibility Insights (opens in new tab) browser plugin to test tab stops. You should also tab through your page to see if elements are selected in the order intended.
