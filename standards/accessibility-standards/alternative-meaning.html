Images and media contain alternative descriptions

Media such as images, video and charts should contain a text alternative describing the media.

This is a requirement of:

WCAG 1.1.1 - Non-text content (Level A)

For images, this is achieved by adding a description using the alt attribute, for example:

<img src='/path-to-image/filename.png' alt='A child in a classroom sitting at a desk listening to a teacher talk' />

For other media like video, this should be done using other means for example:

  • closed captions (subtitles)
  • sign language
  • transcript
  • audio description

When you don’t need to use alternative text

  • The image is purely decorative, like a background pattern
  • The image content is represented by adjacent real text like a product description

You should add an alt attribute but leave it empty.

<img src='/path-to-image/filename.png' alt='' />
Do
Add alt text to images to describe them, this helps screen reader users understand images and describes the image if it fails to load.
Don’t
Missing alt text might make it more difficult for users to understand what is being presented to them.
