Design standards

This guidance should be used alongside the How to apply the Service Standard in DfE manual.

Working to defined standards ensure services the Department for Education (DfE) create are inclusive and accessible and can be used by all people.

This includes removing risk by designing how things should work, this can be achieved by:

  • working with other designers, researchers and developers to build and test prototypes
  • following inclusive design principles
  • ensuring you meet component design criteria
  • contributing to the GOV.UK Design System backlog (opens in new tab)
  • sharing your work with other designers across DfE
  • taking part in design crits, reviews and assessments to assure your designs are meeting these standards

Minimum design standards

The following standards are the minimum required for designers to meet.

Branding the service

How to ensure the service is using the correct header, logos and typeface for where it’s being provided.

Do not use colour alone to convey information

Do not use colour as the only way to explain or signal something.

Make it easy for people to identify interactive elements

Style components such as links and buttons to make them easy to identify in different states.

Design interactions in a logical order

Good tab stop design allows keyboard users to access a services interactive elements in a predictable order.

Make navigation options clear and consistent across the service

It should be easy to get around the service in a consistent way and for users to know where they are in a service.

Include clearly associated labels on form elements

Ensure that all forms have an identifiable label next to, or above the field.

Make sure feedback to the user is clear and identifiable

Tell users clearly if something is happening or they need to take action.

Design for different viewports, ideally mobile-first

Think about how users will interact with the service on different devices and how the design will change as a result.

If you use images and media, make sure they have alternative descriptions

Think about how to describe images, audio descriptions of video or transcripts.

If you use content which starts automatically, make sure you include controls

Allow the user to control or stop any animations or audio.
